## 2026-06-15 - [B324] Use of weak MD5 and SHA1 hash for security
**Vulnerability:** Found multiple instances of `hashlib.md5()` and `hashlib.sha1()` being used without the `usedforsecurity=False` flag. While these are used for non-cryptographic purposes (e.g. caching, chunking, dedup), Bandit flags them as security vulnerabilities because they can be weak or susceptible to collisions when used for security.
**Learning:** Python's `hashlib` requires explicitly setting `usedforsecurity=False` when using weak hashes like MD5 and SHA1 for non-security tasks to avoid triggering security linters and to signal developer intent. The codebase had this pattern repeated across several utility and platform adapter files.
**Prevention:** Always include the `usedforsecurity=False` flag when calling `hashlib.md5()` or `hashlib.sha1()` for caching, hashing large files, deduping content, or generating file IDs. Do not blindly apply it to legitimate cryptographic operations (e.g. `wecom_crypto.py`). Ensure Bandit security linter checks are strictly enforced in pre-commit hooks to catch these omissions.
