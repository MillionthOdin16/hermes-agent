## 2026-06-17 - False Positive Command Injection on Intended Shell Endpoints
**Vulnerability:** Bandit flagged `B602` (Command Injection) on `subprocess.run(shell=True)` calls in `tui_gateway/server.py` within the `shell.exec` and quick command executor methods.
**Learning:** When an endpoint legitimately requires executing arbitrary shell commands (like a terminal gateway utilizing pipes and redirects), replacing `shell=True` with `shlex.split(cmd)` and `shell=False` strips the endpoint of its core functionality and can introduce crashing bugs (e.g. `IndexError` or `ValueError` on empty commands). This results in fixing a "vulnerability" by breaking the feature itself.
**Prevention:** Rather than performing security theater by crippling the endpoint, explicitly acknowledge the design requirement. Ensure robust external safety checks (like `detect_dangerous_command`) are enforced on inputs, and suppress the static analyzer warning using a `# nosec B602` waiver to indicate the risk is accepted and mitigated via application logic.
